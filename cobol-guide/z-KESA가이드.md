# 📑 z-KESA 프레임워크 기반 코볼(COBOL) 상세 개발 가이드

## 1. 프로그램 기본 구조 및 주석 표준

모든 코볼 소스 프로그램은 프레임워크 표준에 따라 시스템 정보와 변경 이력을 문서화해야 합니다. 

### 1.1 필수 주석 항목 (Identification Division 상단)

* 
**프로그램 정보**: 업무명, 프로그램명(한글), 처리 유형, 처리 개요를 반드시 포함해야 합니다. 


* 
**변경 이력**: 변경 근거(CM Package 번호), 변경 일자, 변경 내용, 작성자를 기록합니다. 


* 
**기능 설명**: 각 파라그래프(Paragraph) 상단에 기능 설명, 인수(Argument), 리턴값 정보를 작성합니다. 



## 2. 데이터 정의 및 타입 (Working-Storage Section)

z-KESA 환경에서는 데이터 성격에 따라 다음과 같은 명명 규칙과 패턴을 사용합니다. 

| 타입 | 설명 | 정의 패턴 (Pattern) | 예시 |
| --- | --- | --- | --- |
| **C (Character)** | 문자형 | `'C' + [길이]` | `PIC X(10)` (길이 10인 문자) |
| **N (Numeric)** | 숫자형 | `N + 0 + [부호] + [전체길이] + [소수길이]` | `PIC S9(07)V9(02)` (부호 포함 전체 9자리, 소수 2자리) |

## 3. 핵심 인터페이스: 카피북(Copybook) 관리

* 
**자동 생성**: AS 프로그램의 입출력 카피북, DBIO/SQLIO 인터페이스 구조체는 BIM(카피북 관리 시스템)을 통해 자동 생성됩니다. 


* 
**수동 보완**: 자동 생성된 카피북에 개발자가 직접 필요한 주석을 추가하여 가독성을 높여야 합니다. 



## 4. DB 및 외부 프로그램 호출 매크로

프레임워크에서 제공하는 전용 매크로를 사용하여 복잡한 로직을 간소화합니다. 

* 
**#SQLIO**: 복잡한 SQL 처리 및 Multi-Row Fetch(다건 조회) 시 사용합니다. 


* 
**#DYCALL**: 다른 프로그램을 동적으로 호출할 때 사용하며, 파라미터 전달 방식을 규정합니다. 


* 
**#SCRENO**: 출력 화면 번호를 지정할 때 사용합니다. 


* 
**#GETNCS**: 시스템 일련번호(채번)가 필요할 때 호출합니다. 



## 5. 공통 정보 영역 (Common Area) 활용 상세

프로그램 실행 시 모든 계층에서 참조 가능한 데이터 영역입니다. 

| 영역 명칭 | 주요 내용 | 비고 |
| --- | --- | --- |
| **BICOM** | 입력 내부표준전문 공통부 정보 | 거래 기본 입력 정보 

 |
| **BOCOM** | 출력 결과 및 프레임워크 제어 정보 | 출력 폼, 에러 정보 등 

 |
| **JICOM** | 전행 공통 정보 | 일자, 부점 정보 등 

 |
| **AACOM** | 고객 정보 | 고객 기본 및 세부 정보 

 |
| **DMCOM** | 상품 정보 | 계좌의 상품 기본 정보 

 |
| **HCCOM** | 계약 정보 | 호스트 파일 참조 정보 

 |

## 6. 오류 처리 및 유틸리티

* 
**에러 전문 조립**: 대외거래나 온라인 거래 실패 시 프레임워크에서 제공하는 에러 처리 프로그램(ZUGCDCV 등)을 통해 표준화된 에러 메시지를 생성합니다. 


* 
**SAF (Strong Authentication Framework)**: 특정 상황에서 강제로 메시지를 처리하거나 에러를 통제할 때 사용합니다. 


* **코드 변환 유틸리티**:
* 
**ZUDAV03 / ZUGCDCV**: ASCII ↔ EBCDIC 간 코드 변환 수행. 


* 
**ZUDAV20**: HEX ↔ CHAR 간 코드 변환 수행.
