## 1. 개요
본 문서는 기업집단신용평가이력관리 시스템에 대한 포괄적인 테스트 케이스를 정의합니다. 테스트 케이스는 업무 명세서에 정의된 모든 업무 기능, 엔티티, 규칙을 다루며, 12개 관련 데이터베이스 테이블에 걸친 기업집단 신용평가 레코드의 CRUD 작업에 대한 완전한 검증을 보장하고 긍정적, 부정적, 경계 시나리오의 포괄적인 커버리지를 제공합니다.

## 2. 테스트 구성
테스트는 기능 우선순위와 처리 유형별로 구성됩니다:
- **핵심 기능**: 신규평가 생성, 평가삭제, 입력 검증
- **지원 기능**: 직원정보 조회, 주채무계열여부 조회, 중복평가 확인
- **처리 유형**: '01' (신규평가), '03' (평가삭제)
- **테스트 유형**: 긍정적 (정상 흐름), 부정적 (오류 조건), 경계 (한계 테스트)

## 3. 테스트 우선순위
- **핵심**: 핵심 CRUD 작업 및 데이터 무결성 검증
- **높음**: 입력 검증 및 업무 규칙 적용
- **보통**: 직원 및 부점 정보 조회
- **낮음**: 기본값 할당 및 시스템 컨텍스트 처리

## 4. 테스트 의존성
- THKIPB110-THKIPB133 테스트 데이터를 포함한 데이터베이스 설정
- 검증을 위한 직원 마스터 데이터 (THKIPA302)
- 검증을 위한 부점 마스터 데이터 (THKJIBR01)
- 유효한 그룹회사코드를 포함한 시스템 구성
- 검증 테스트를 위한 오류코드 구성

## 5. 테스트 케이스

### TC-WP052-001: 유효한 신규 기업집단 신용평가 생성 (Valid New Corporate Group Credit Evaluation Creation)
- **유형:** 긍정적
- **우선순위:** 핵심
- **설명:** 모든 필수 매개변수와 적절한 기본값 초기화를 통한 신규 기업집단 신용평가의 성공적인 생성 테스트
- **기능:** F-052-001, F-052-003, F-052-004, F-052-005, F-052-006
- **엔티티:** BE-052-001, BE-052-002, BE-052-003, BE-052-004
- **전제조건:**
  - 시스템이 유효한 그룹회사코드로 초기화됨
  - 직원 마스터에 현재 사용자 정보 포함
  - 동일한 키에 대한 기존 확정평가 없음
  - THKIPB110 테이블이 삽입 작업에 접근 가능
- **테스트 데이터:**
  - **입력:**
    - 처리구분코드: "01"
    - 기업집단그룹코드: "001"
    - 기업집단명: "KB금융그룹"
    - 평가년월일: "20241201"
    - 평가기준년월일: "20241130"
    - 기업집단등록코드: "001"
  - **예상 출력:**
    - 총건수: 1
    - 현재건수: 1
    - 모든 기본값을 포함한 THKIPB110 레코드 생성
- **테스트 단계:**
  1. 테스트 매개변수로 시스템 초기화
  2. 업무 규칙에 따른 모든 입력 매개변수 검증
  3. 중복 확정평가 레코드 확인
  4. 현재 사용자에 대한 직원정보 조회
  5. 기업집단에 대한 주채무계열여부 조회
  6. 기본값으로 THKIPB110 레코드 초기화
  7. 데이터베이스에 신규평가 레코드 삽입
  8. 성공적인 생성 확인 및 응답 반환
- **검증 포인트:**
  - 모든 입력 검증 규칙 통과 (BR-052-001 ~ BR-052-005)
  - 중복 확정평가 존재하지 않음 (BR-052-006)
  - 직원정보가 올바르게 조회됨 (BR-052-009)
  - 주채무계열여부가 적절히 결정됨 (BR-052-010)
  - 기본값이 올바르게 할당됨 (BR-052-008, BR-052-015, BR-052-016, BR-052-017)
  - 시스템 컨텍스트 값이 적절히 설정됨 (BR-052-007, BR-052-013, BR-052-014)
  - 데이터베이스 레코드가 성공적으로 생성됨
- **의존성:** 없음

### TC-WP052-002: 유효한 기업집단 신용평가 삭제 (Valid Corporate Group Credit Evaluation Deletion)
- **유형:** 긍정적
- **우선순위:** 핵심
- **설명:** 모든 12개 관련 테이블에 걸친 연쇄 작업을 통한 기업집단 신용평가의 성공적인 삭제 테스트
- **기능:** F-052-002
- **엔티티:** BE-052-001, BE-052-002, BE-052-003
- **전제조건:**
  - 시스템이 유효한 그룹회사코드로 초기화됨
  - THKIPB110 및 관련 테이블에 기존 평가 레코드 존재
  - 데이터베이스 테이블이 삭제 작업에 접근 가능
- **테스트 데이터:**
  - **입력:**
    - 처리구분코드: "03"
    - 기업집단그룹코드: "002"
    - 기업집단명: "신한금융그룹"
    - 평가년월일: "20241115"
    - 기업집단등록코드: "002"
  - **예상 출력:**
    - 총건수: 1
    - 현재건수: 1
    - 12개 테이블에서 모든 관련 레코드 삭제
- **테스트 단계:**
  1. 테스트 매개변수로 시스템 초기화
  2. 업무 규칙에 따른 모든 입력 매개변수 검증
  3. 순서대로 연쇄 삭제 작업 실행
  4. THKIPB110 (기업집단평가기본) 삭제
  5. THKIPB111 (기업집단연혁명세) 삭제
  6. THKIPB116 (기업집단계열사명세) 삭제
  7. THKIPB113 (기업집단사업부분구조분석명세) 삭제
  8. THKIPB112 (기업집단재무분석목록) 삭제
  9. THKIPB114 (기업집단항목별평가목록) 삭제
  10. THKIPB118 (기업집단평가등급조정사유목록) 삭제
  11. THKIPB130 (기업집단주석명세) 삭제
  12. THKIPB131 (기업집단승인결의록명세) 삭제
  13. THKIPB132 (기업집단승인결의록위원명세) 삭제
  14. THKIPB133 (기업집단승인결의록의견명세) 삭제
  15. THKIPB119 (기업집단재무평점단계별목록) 삭제
  16. 성공적인 삭제 확인 및 응답 반환
- **검증 포인트:**
  - 모든 입력 검증 규칙 통과 (BR-052-001 ~ BR-052-004)
  - 연쇄 삭제 순서가 올바르게 따라짐 (BR-052-018)
  - 락 기반 레코드 접근 사용 (BR-052-012)
  - 커서 기반 배치 처리 적용 (BR-052-019)
  - 삭제 전 데이터 존재 확인 (BR-052-020)
  - 모든 관련 레코드가 성공적으로 삭제됨
- **의존성:** TC-WP052-001
