## 1. 개요

본 워크패키지는 신용평가 및 고객관리 목적으로 기업집단 합산재무제표를 생성하는 포괄적인 배치 처리 시스템을 구현합니다. 시스템은 기업집단 평가 데이터를 처리하고 계열사들의 개별재무제표를 다년도에 걸쳐 집계하여 합산재무제표를 생성합니다.

업무 목적은 다음과 같습니다:
- 기업집단 합산재무제표 생성
- 계열사 개별재무제표 집계
- 다년도 결산년 처리를 통한 포괄적 재무분석
- 합산재무지표를 통한 신용평가 지원
- 추세분석을 위한 과거 재무데이터 유지
- 당행 및 외부평가기관 재무데이터 처리

시스템은 확장된 다중 모듈 흐름을 통해 데이터를 처리합니다: BIP0021 → RIPA120 → YCCOMMON → YCDBIOCA → YCCSICOM → YCCBICOM → TRIPC120 → TKIPC120 → TRIPC140 → TKIPC140, 기업집단 재무데이터 집계, 개별재무제표 처리, 합산재무제표 생성을 담당합니다.

주요 업무 기능은 다음을 포함합니다:
- 기업집단 식별 및 선택
- 다년도 재무데이터 처리 (4년 과거 데이터: 당해년도부터 4년 전까지)
- 내부 및 외부 소스로부터 개별재무제표 수집
- 합산재무제표 계산 및 생성
- 임시 개별재무제표 관리
- 데이터 무결성 및 감사추적 유지

| 기업집단그룹코드 | String | 3 | NOT NULL | 기업집단 분류 코드 | WK-DB-CORP-CLCT-GROUP-CD | corpClctGroupCd |

## 3. 업무 규칙

### BR-009-001: 기업집단선택규칙
- **설명:** 합산재무제표 생성 대상 기업집단 선택 규칙
- **조건:** THKIPB110에 기업집단이 존재하고 THKIPC110에 재무제표반영여부 = '0'이며 기업집단처리단계구분 != '6'인 경우 처리 대상으로 선택
- **관련 엔티티:** [BE-009-001]
- **예외사항:** 기업집단처리단계구분 = '6' (완료된 처리 단계)인 그룹은 제외

### BR-009-002: 다년도처리규칙
- **설명:** 다년도(4년 범위)에 걸친 재무제표 처리 규칙
- **조건:** 기업집단이 선택되면 4년 전부터 당해 기준년까지 각 년도를 반복하여 재무데이터를 수집하고 합산
- **관련 엔티티:** [BE-009-001, BE-009-002]
- **예외사항:** 4년 전의 경우 데이터 가용성으로 인해 계열사 선택에 3년 전 데이터 사용

### BR-009-003: 개별재무제표우선순위규칙
- **설명:** 재무제표 소스 우선순위 규칙 (당행 vs 외부평가기관)
- **조건:** 계열사 처리 시 먼저 당행 재무제표(THKIIK319) 확인, 없으면 외부평가기관 데이터(THKIIMA10)를 자료원구분 '2' 우선, '3' 차순으로 확인
- **관련 엔티티:** [BE-009-003]
- **예외사항:** 계산 날짜 범위 내에서 가장 최근 데이터 사용

### BR-009-004: 재무데이터검증규칙
- **설명:** 합산 전 재무데이터 검증 규칙
- **조건:** 재무항목금액이 0이 아닌 경우 합산에 포함, 0인 경우 성능 최적화를 위해 처리에서 제외
- **관련 엔티티:** [BE-009-003, BE-009-004]
- **예외사항:** 0 금액은 유효하지만 효율성을 위해 처리하지 않음

### BR-009-005: 합산재무제표생성규칙
- **설명:** 개별재무제표를 합산하여 합산재무제표 생성 규칙
- **조건:** 결산년도별로 모든 개별재무제표가 수집되면 재무항목코드와 보고서구분별로 금액을 합산하여 기업집단, 결산년, 재무항목별로 그룹화
- **관련 엔티티:** [BE-009-004]
- **예외사항:** NULL 값은 합산에서 0으로 처리

### BR-009-006: 기존데이터정리규칙
- **설명:** 새 데이터 생성 전 기존 합산재무제표 데이터 정리 규칙
- **조건:** 기업집단의 합산 프로세스 시작 시 중복 방지를 위해 동일한 기준년의 기존 합산재무제표(THKIPC120) 삭제
- **관련 엔티티:** [BE-009-004]
- **예외사항:** 처리 중인 특정 기업집단과 기준년의 데이터만 삭제

### BR-009-007: 임시데이터관리규칙
- **설명:** 임시 개별재무제표 데이터(THKIPC140) 관리 규칙
- **조건:** 개별재무제표 처리 시 합산을 위해 임시 테이블(THKIPC140)에 저장, 합산 완료 후 선택적으로 임시 데이터 정리
- **관련 엔티티:** [BE-009-003]
- **예외사항:** 임시 데이터 정리는 설정 가능하며 감사 목적으로 비활성화될 수 있음

### BR-009-008: 입력파라미터검증규칙
- **설명:** 배치 입력 파라미터 검증 규칙
- **조건:** 배치 시작 시 작업기준년월일이 공백이 아닌지 검증, 공백이면 오류코드 11로 종료
- **관련 엔티티:** [BE-009-001]
- **예외사항:** 작업기준년월일은 배치 처리에 필수

### BR-009-009: 오류처리규칙
- **설명:** 처리 중 다양한 오류 조건 처리 규칙
- **조건:** 오류 발생 시 적절한 반환 코드 설정: 입력 파라미터 오류 11-19, 데이터베이스 오류 21-29, 파일 제어 오류 91-99, 그리고 처리 종료
- **관련 엔티티:** [BE-009-001, BE-009-002, BE-009-003, BE-009-004]
- **예외사항:** 오류 유형별로 다른 오류 코드 사용

### BR-009-010: 날짜범위계산규칙
- **설명:** 재무데이터 선택을 위한 날짜 범위 계산 규칙
- **조건:** 결산년 처리 시 데이터 선택을 위해 (계산기준일 - 1년 + 1일)부터 계산기준일까지의 날짜 범위 계산
- **관련 엔티티:** [BE-009-002, BE-009-003]
- **예외사항:** 윤년과 월말 날짜를 올바르게 처리

## 4. 업무 기능

### F-009-001: 기업집단선택기능
- **설명:** 합산재무제표 생성 대상 기업집단 선택
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 그룹회사코드 | String | 3 | Fixed='KB0' | KB금융그룹 식별자 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 기업집단그룹코드 | String | 3 | NOT NULL | 선택된 기업집단 코드 |
| 기업집단등록코드 | String | 3 | NOT NULL | 선택된 기업집단 등록 |
| 기준년월 | String | 6 | YYYYMM 형식 | 처리를 위한 기준년월 |

- **처리 로직:**
  - 기업집단처리단계구분 != '6'인 기업집단을 THKIPB110에서 조회
  - 재무제표반영여부 = '0'인 그룹을 찾기 위해 THKIPC110과 조인
  - 그룹회사코드 = 'KB0'로 필터링
  - 처리를 위한 고유한 기업집단 식별자 반환
- **적용 업무규칙:** [BR-009-001]

### F-009-002: 다년도날짜계산기능
- **설명:** 다년도 재무제표 처리를 위한 날짜 범위 계산
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 기준년월 | String | 6 | YYYYMM 형식 | 계산을 위한 기준년월 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 기준년 | String | 4 | YYYY 형식 | 계산된 기준년 |
| 4년기준년월일 | Date | 8 | YYYYMMDD 형식 | 4년 전 날짜 |
| 기준년월일 | Date | 8 | YYYYMMDD 형식 | 처리를 위한 기준일 |

- **처리 로직:**
  - LAST_DAY 함수를 사용하여 기준년월을 해당 월의 마지막 날로 변환
  - ADD_MONTHS(-12*3) 함수를 사용하여 4년 전 날짜 계산
  - 반복 제어를 위한 년도 구성요소 추출
  - 각 결산년 처리를 위한 날짜 범위 설정
- **적용 업무규칙:** [BR-009-002, BR-009-010]

### F-009-003: 계열사선택기능

### F-009-004: 당행재무제표처리기능
- **설명:** 계열사의 당행 재무제표 처리
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 심사고객식별자 | String | 10 | NOT NULL | 고객 식별자 |
| 계산기준년월일 | Date | 8 | YYYYMMDD 형식 | 데이터 선택을 위한 기준일 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 신용평가보고서번호 | String | 13 | NOT NULL | 보고서 식별자 |
| 결산기준년월일 | Date | 8 | YYYYMMDD 형식 | 결산일 |
| 재무제표존재여부 | String | 1 | 'Y'/'N' | 존재 플래그 |

- **처리 로직:**
  - 날짜 범위 내에서 신용평가보고서를 THKIIK616에서 조회
  - THKIIK319에서 재무제표 데이터 확인
  - 데이터 최신성과 완전성 검증
  - 재무제표 가용성과 식별자 반환
- **적용 업무규칙:** [BR-009-003, BR-009-010]

### F-009-005: 외부재무제표처리기능
- **설명:** 당행 데이터가 없을 때 외부평가기관 재무제표 처리
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 심사고객식별자 | String | 10 | NOT NULL | 고객 식별자 |
| 계산기준년월일 | Date | 8 | YYYYMMDD 형식 | 데이터 선택을 위한 기준일 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 재무분석자료원구분 | String | 1 | '2','3' | 외부 자료원 구분 |
| 결산기준년월일 | Date | 8 | YYYYMMDD 형식 | 결산일 |
| 재무분석결산구분 | String | 1 | Fixed='1' | 결산 구분 |

- **처리 로직:**
  - 재무분석자료번호 구성: '07' + 고객식별자
  - 우선순위로 외부평가 데이터를 THKIIMA10에서 조회
  - 날짜 범위 내에서 가장 최근 데이터 선택
  - 자료원구분 '2'를 '3'보다 우선
- **적용 업무규칙:** [BR-009-003, BR-009-010]

### F-009-006: 개별재무제표저장기능
- **설명:** 합산을 위해 개별재무제표 데이터를 임시 테이블에 저장
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 재무제표데이터 | Record | Variable | NOT NULL | 완전한 재무제표 레코드 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 저장상태 | String | 2 | '00'=성공 | 저장 작업 결과 |
| 레코드수 | Numeric | 9 | 양의 정수 | 저장된 레코드 수 |

- **처리 로직:**
  - 재무제표 데이터를 THKIPC140 (임시 테이블)에 삽입
  - 데이터 무결성과 제약조건 검증
  - 중복 키 상황 처리
  - 통계를 위한 성공적인 삽입 수 계산

| 결산년 | String | 4 | YYYY 형식 | 합산을 위한 결산년 |

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|

```
기업집단 합산재무제표 생성 프로세스 흐름

1. 초기화 단계
   ├── SYSIN 파라미터 수신 (작업기준년월일)
   ├── 입력 파라미터 검증
   └── 처리 변수 초기화

2. 기업집단 선택 단계
   ├── 대상 기업집단을 THKIPB110에서 조회
   ├── 재무제표반영상태를 위해 THKIPC110과 조인
   ├── 처리단계와 반영상태로 필터링
   └── 처리할 기업집단 선택

3. 다년도 처리 단계 (각 기업집단별)
   ├── 기준년과 4년 날짜 범위 계산
   ├── 기존 합산재무제표 삭제
   └── 각 년도별 처리 (4년 전부터 당해년도까지)
       ├── 년도별 날짜 범위 계산
       ├── 해당 년도의 계열사 선택
       └── 개별재무제표 처리

4. 개별재무제표 처리 (각 계열사별)
   ├── 당행 재무제표 확인 (THKIIK616/THKIIK319)
   ├── 당행 데이터가 존재하는 경우:
   │   ├── 재무제표 데이터 추출
   │   └── 임시 테이블(THKIPC140)에 저장
   ├── 그렇지 않으면 외부평가기관 데이터 확인 (THKIIMA10)
   │   ├── 우선순위로 외부 재무데이터 조회
   │   └── 임시 테이블(THKIPC140)에 저장
   └── 처리된 재무제표 수 계산

5. 합산 단계 (각 결산년별)
   ├── THKIPC140에서 모든 개별재무제표 조회
   ├── 재무항목코드와 보고서구분별로 그룹화
   ├── 재무제표항목금액 합산
   ├── 업체 수 통계 계산
   └── 합산 결과를 THKIPC120에 삽입

6. 정리 단계 (선택사항)
   ├── 임시 개별재무제표(THKIPC140) 정리
   └── 모든 트랜잭션 커밋

7. 완료 단계
   ├── 처리 통계 생성
   ├── 완료 상태 로그
   └── 처리 결과 반환
```

### 업무규칙 구현
- **BR-009-001:** BIP0021.cbl 200-230행에 구현 (CUR_C001 커서)
  ```cobol

  FROM DB2DBA.THKIPB110 B110, (SELECT DISTINCT 그룹회사코드, 기업집단그룹코드, 기업집단등록코드
  FROM DB2DBA.THKIPC110 WHERE 그룹회사코드 = 'KB0' AND 재무제표반영여부 = :CO-NO) C110
  WHERE B110.그룹회사코드 = C110.그룹회사코드 AND B110.기업집단처리단계구분 != '6'
  ```

### 기술 아키텍처
- **BATCH Layer**: BIP0021 - JCL 작업 제어가 포함된 메인 배치 처리 프로그램
- **SQLIO Layer**: RIPA120 - THKIPA120 테이블 조작을 위한 데이터베이스 I/O 컴포넌트
- **Framework**: YCCOMMON, YCDBIOCA, YCCSICOM, YCCBICOM - 트랜잭션 관리, 데이터베이스 조작, 업무 로직을 위한 공통 프레임워크 컴포넌트

### 데이터 흐름 아키텍처
1. **입력 흐름**: SYSIN → BIP0021 → 파라미터 검증 → 기업집단 선택
2. **데이터베이스 접근**: BIP0021 → RIPA120 → THKIPB110/THKIPC110/THKIIK319/THKIIK616/THKIIMA10
3. **처리 흐름**: 개별재무제표 수집 → 임시 저장 (THKIPC140) → 합산 → 최종 저장 (THKIPC120)
4. **출력 흐름**: 합산재무제표 → THKIPC120 → 처리 통계 → 배치 완료
5. **오류 처리**: 모든 레이어 → 프레임워크 오류 처리 → 오류 코드 → 배치 종료
