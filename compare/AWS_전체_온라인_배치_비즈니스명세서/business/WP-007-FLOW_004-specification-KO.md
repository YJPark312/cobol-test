# 업무 명세서: 기업집단 합산현금수지생성

## 1. 개요

본 워크패키지는 신용평가 및 보고 목적을 위한 기업집단 합산현금수지표 생성 배치 처리 시스템을 구현합니다. 시스템은 기업집단 평가 데이터를 처리하고 여러 결산년도 및 기업집단 엔티티에 걸쳐 복잡한 재무 공식을 계산하여 합산현금수지표를 생성합니다.

업무 목적:
- 기업집단 합산현금수지표 생성
- THKIIMB11(재무분석항목기본)의 사전 정의된 공식을 사용한 현금수지 항목 계산
- 포괄적인 현금수지 분석을 위한 다년도 결산년 처리
- 합산현금수지지표를 통한 신용평가 지원
- 추세분석을 위한 과거 현금수지 데이터 유지

시스템은 복잡한 다중 모듈 흐름을 통해 데이터를 처리합니다: BIP0022 → RIPA120 → YCCOMMON → YCDBIOCA → YCCSICOM → YCCBICOM → TRIPC120 → TKIPC120, 기업집단 재무 데이터 집계 및 현금수지 계산을 처리합니다.

## 3. 업무 규칙

### BR-007-001: 기업집단선택규칙
- **설명:** 합산현금수지 생성 대상 기업집단 선택 규칙
- **조건:** THKIPB110에 기업집단이 존재하고 THKIPC110에 재무제표반영여부 = '0'인 해당 항목이 있으며 기업집단처리단계구분 != '6'인 경우 처리 대상으로 선택
- **관련 엔티티:** [BE-007-001]
- **예외사항:** 기업집단처리단계구분 = '6'(완료된 처리 단계)인 그룹은 제외

### BR-007-002: 결산년처리규칙
- **설명:** 각 기업집단에 대한 다중 결산년 처리 규칙
- **조건:** 기업집단이 선택되면 THKIPC120에서 재무분석결산구분 = '1'이고 기준년이 일치하는 모든 고유 결산년을 처리
- **관련 엔티티:** [BE-007-001, BE-007-002]
- **예외사항:** THKIPC120에 유효한 데이터가 있는 결산년만 처리

### BR-007-003: 기존데이터삭제규칙
- **설명:** 재생성 전 기존 현금수지 데이터 삭제 규칙
- **조건:** 기업집단 처리 시 THKIPC120에서 재무분석보고서구분 = '18'(현금수지 분류)이고 그룹/등록코드 및 기준년이 일치하는 모든 기존 레코드를 삭제
- **관련 엔티티:** [BE-007-004]
- **예외사항:** 데이터 손실을 방지하기 위해 정확한 조건에 일치하는 레코드만 삭제

### BR-007-004: 산식처리규칙
- **설명:** 현금수지 항목 계산을 위한 재무 공식 처리 규칙
- **조건:** 공식에 계산 표현식이 포함된 경우 THKIPC120의 실제 값으로 변수를 치환하고 수학적 표현식을 평가
- **관련 엔티티:** [BE-007-003]
- **예외사항:** NULL 값을 0으로 처리, 잘못된 연산 대체

### BR-007-005: 값범위검증규칙
- **설명:** 계산된 현금수지 값의 허용 범위 내 검증 규칙
- **조건:** 계산값 > 999999999999999인 경우 999999999999999로 설정, 계산값 < -999999999999999인 경우 -999999999999999로 설정
- **관련 엔티티:** [BE-007-003, BE-007-004]
- **예외사항:** 오버플로우 오류 방지를 위한 범위 제한 적용

### BR-007-006: 자료원구분규칙
- **설명:** 합산현금수지 데이터의 자료원 구분 설정 규칙
- **조건:** 합산현금수지 데이터 입력 시 재무분석자료원구분 = 'S'(합산 자료원)로 설정
- **관련 엔티티:** [BE-007-004]
- **예외사항:** 합산현금수지표의 경우 항상 'S' 사용

### BR-007-007: 배치처리제어규칙
- **설명:** 배치 처리 흐름 및 오류 처리 제어 규칙
- **조건:** 처리 오류 발생 시 적절한 리턴 코드 설정(11-19: 입력 오류, 21-29: DB 오류, 91-99: 파일 제어 오류) 후 처리 종료
- **관련 엔티티:** [BE-007-001, BE-007-002, BE-007-003, BE-007-004]
- **예외사항:** 오류 유형별로 다른 오류 코드 적용

### BR-007-008: 산식변수치환규칙
- **설명:** 계산 공식의 변수를 실제 재무 데이터로 치환하는 규칙
- **조건:** 공식에 '&' 변수가 포함된 경우 재무분석보고서구분, 재무항목코드, 결산년 지시자('C': 당년, 'B': 전년)를 기반으로 THKIPC120의 해당 값으로 대체
- **관련 엔티티:** [BE-007-003]
- **예외사항:** 누락된 데이터를 0으로 처리, 중첩된 공식 참조 처리

### BR-007-009: 커밋처리규칙
- **설명:** 데이터베이스 트랜잭션 커밋 처리 규칙
- **조건:** 결산년별 모든 현금수지 계산이 완료되면 데이터 일관성 보장을 위해 트랜잭션 커밋
- **관련 엔티티:** [BE-007-004]
- **예외사항:** 처리 오류 시 롤백

## 4. 업무 기능

### F-007-001: 기업집단선택기능
- **설명:** 평가 기준에 따라 합산현금수지 생성 대상 기업집단을 선택
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 그룹회사코드 | String | 3 | Fixed='KB0' | KB금융그룹 식별자 |
| 처리단계필터 | String | 1 | != '6' | 완료된 처리 단계 제외 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 기업집단그룹코드 | String | 3 | NOT NULL | 선택된 기업집단 코드 |
| 기업집단등록코드 | String | 3 | NOT NULL | 선택된 등록 코드 |
| 기준년월 | String | 6 | YYYYMM | 처리 기준 기간 |

- **처리 로직:**
  - 그룹회사코드 = 'KB0'인 기업집단에 대해 THKIPB110 조회
  - 재무제표반영여부 = '0'인 그룹을 필터링하기 위해 THKIPC110과 조인
  - 기업집단처리단계구분 = '6'인 그룹 제외
  - 고유한 기업집단 식별자 및 기준년월 반환
- **적용 업무규칙:** [BR-007-001]

### F-007-002: 결산년처리기능
- **설명:** 선택된 기업집단의 모든 결산년을 처리하여 현금수지 데이터 생성
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 기업집단그룹코드 | String | 3 | NOT NULL | 대상 기업집단 |
| 기업집단등록코드 | String | 3 | NOT NULL | 대상 등록 코드 |
| 기준년 | String | 4 | YYYY | 처리 기준년 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 결산년 | String | 4 | YYYY | 처리된 결산년 |
| 결산년합계업체수 | Numeric | 9 | 양수 | 해당 년도 회사 수 |
| 처리상태 | String | 2 | '00'=성공 | 처리 결과 상태 |

- **처리 로직:**
  - 재무분석결산구분 = '1'인 고유 결산년에 대해 THKIPC120 조회
  - 기업집단 코드 및 기준년으로 필터링
  - 결산년 내림차순 정렬
  - 전년도(결산년B)를 당년도에서 1을 뺀 값으로 계산
  - 각 결산년에 대해 현금수지 생성 처리
- **적용 업무규칙:** [BR-007-002]

  - 감사 추적을 위한 삭제 레코드 수 계산
  - 성공적인 삭제 후 트랜잭션 커밋
- **적용 업무규칙:** [BR-007-003, BR-007-009]

- **입력:**

- **BATCH Layer**: BIP0022 - 기업집단 현금수지 생성을 위한 JCL 작업 제어가 포함된 주요 배치 처리 프로그램
- **SQLIO Layer**: RIPA120, TRIPC120, TKIPC120 - THKIPC120 테이블 작업을 위한 데이터베이스 액세스 컴포넌트
- **Framework**: YCCOMMON, YCDBIOCA, YCCSICOM, YCCBICOM - 시스템 통합 및 데이터베이스 작업을 위한 공통 프레임워크 컴포넌트

### 데이터 흐름 아키텍처

1. **입력 흐름**: SYSIN 매개변수 → BIP0022 → 매개변수 검증 → 기업집단 선택
2. **데이터베이스 액세스**: BIP0022 → YCDBIOCA → THKIPB110/THKIPC110/THKIPC120 → 공식 데이터 조회
3. **서비스 호출**: BIP0022 → XFIIQ001 (재무산식파싱) → 수학적 계산 결과
4. **출력 흐름**: 계산 결과 → YCDBIOCA → THKIPC120 → 합산현금수지 데이터 저장
5. **오류 처리**: 모든 계층 → 프레임워크 오류 처리 → 배치 작업 상태 코드 → JCL 오류 처리
