## 1. 개요

이 워크패키지는 기업집단차주변동명세를 추출하는 배치 처리 시스템을 구현합니다. 시스템은 일일 기업집단차주 변경 데이터를 처리하고 신용평가 목적으로 암호화된 출력 파일을 생성합니다. 주요 흐름은 BIP0013(기업집단차주변동명세)에서 YCCOMMON(공통처리모듈)로 데이터를 처리합니다.

업무 목적은 다음과 같습니다:
- 기업집단차주 일일변동내역 추출
- 데이터 보안을 위한 고객고유번호 암호화 처리
- 신용평가용 표준변동명세서 생성
- BT지주리스크 기업신용평가 자료제공

## 3. 업무 규칙

### BR-004-001: 그룹회사코드검증
- **설명:** 모든 처리가 KB금융그룹으로 제한되도록 검증
- **조건:** 레코드 처리 시 그룹회사코드는 'KB0'과 같아야 함
- **관련 엔티티:** [BE-004-001]
- **예외:** 그룹회사코드가 'KB0'이 아니면 처리 종료

### BR-004-002: 고객고유번호구분필터
- **설명:** 특정 고객고유번호구분을 처리에서 제외
- **조건:** 고객고유번호구분이 '01'과 같으면 레코드를 처리에서 제외
- **관련 엔티티:** [BE-004-001]
- **예외:** 구분이 '01'인 레코드는 필터링되어 처리되지 않음

### BR-004-003: 기업집단등록구분할당
- **설명:** 기업집단 변경 레코드에 고정 등록구분 할당
- **조건:** 기업집단 변경 레코드 처리 시 기업집단등록구분을 '3'으로 설정
- **관련 엔티티:** [BE-004-001]
- **예외:** 예외 없음 - 모든 레코드는 구분 '3'을 받음

### BR-004-004: 수기변경구분필터
- **설명:** 현재 및 이력 데이터에 대한 수기변경구분 기준으로 레코드 필터링
- **조건:** 현재 데이터 조회 시 수기변경구분은 '2'이고 등록변경거래구분은 '2'여야 함
- **관련 엔티티:** [BE-004-001]
- **예외:** 이 조건에 맞지 않는 레코드는 현재 데이터 추출에서 제외

### BR-004-005: 이력데이터조회규칙
- **설명:** 변경 전 기업집단 정보를 위한 이력 데이터 조회
- **조건:** 이력 데이터 조회 시 수기변경구분은 '3'이고 등록변경거래구분은 '2'여야 함
- **관련 엔티티:** [BE-004-001]
- **예외:** 이 조건에 맞지 않는 이력 레코드는 제외

### BR-004-006: 기준일자필터규칙
- **설명:** 시스템최종처리일시 기준으로 레코드 필터링
- **조건:** 시스템최종처리일시가 기준일자 이상이면 처리에 포함
- **관련 엔티티:** [BE-004-001, BE-004-002]
- **예외:** 처리일시가 기준일자 이전인 레코드는 제외

### BR-004-007: 고객고유번호암호화규칙
- **설명:** 고객고유번호가 존재할 때 암호화 적용
- **조건:** 고객고유번호가 비어있지 않으면 ASCII 변환 후 일방향 암호화 서비스를 사용하여 암호화
- **관련 엔티티:** [BE-004-001, BE-004-003]
- **예외:** 빈 고객고유번호는 암호화 없이 공백으로 유지

### BR-004-008: 레코드암호화규칙
- **설명:** 양방향 암호화를 사용하여 전체 출력 레코드 암호화
- **조건:** 출력 레코드 작성 시 ASCII 변환 후 양방향 암호화 적용
- **관련 엔티티:** [BE-004-001, BE-004-003]
- **예외:** 암호화 실패는 카운트되지만 처리는 계속됨

### BR-004-009: 작업기준일자기본값규칙
- **설명:** 입력 파라미터에 제공되지 않을 때 기본 작업기준일자 설정
- **조건:** 작업기준일자가 빈값이거나 '00000000'이면 시스템 현재일자에서 1일을 뺀 값으로 설정
- **관련 엔티티:** [BE-004-002]
- **예외:** 시스템 날짜 조회 실패 시 오류로 처리 종료

### BR-004-010: 암호화서비스선택규칙
- **설명:** 시스템 분류에 따른 적절한 암호화 서비스 선택
- **조건:** 시스템구분이 'ZAD'이면 'KB0KIID01'/'KB0KIID02' 사용, 'ZAB'이면 'KB0KIIB01'/'KB0KIIB02' 사용, 'ZAP'이면 'KB0KIIP01'/'KB0KIIP02' 사용, 그 외에는 'KB0KIIB01'/'KB0KIIB02' 사용
- **관련 엔티티:** [BE-004-002, BE-004-003]
- **예외:** 암호화 서비스 구성을 찾을 수 없으면 오류로 처리 종료

## 4. 업무 기능

### F-004-001: 기업집단변경데이터추출
- **설명:** 데이터베이스 테이블에서 기업집단차주 변경 레코드 추출
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|-----------|-----------|--------|-------------|-------------|
| 작업기준년월일 | Date | 8 | YYYYMMDD format | 데이터 추출의 기준 날짜 |
| 그룹회사코드 | String | 3 | Fixed='KB0' | 그룹회사 식별자 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|-----------|-----------|--------|-------------|-------------|
| 기업집단변경기록 | Record Set | Variable | Multiple records | 조건에 맞는 변경 레코드 집합 |

- **처리 로직:**
  - THKIPA112와 THKAABPCB 테이블을 조인하는 복합 SQL 쿼리 실행
  - 그룹회사코드 'KB0'으로 레코드 필터링
  - 고객고유번호구분 '01' 제외
  - 수기변경구분 필터 적용 (현재는 '2', 이력은 '3')
  - 시스템최종처리일시 >= 기준일자로 필터링
  - 현재 및 이력 데이터를 조인하여 완전한 변경 레코드 생성
  - 현재 데이터에 대해 고객별 최대 등록일시 추출
  - 변경 전후 정보를 단일 레코드로 결합
- **적용된 업무 규칙:** [BR-004-001, BR-004-002, BR-004-004, BR-004-005, BR-004-006]

### F-004-002: 고객고유번호암호화처리
- **설명:** 데이터 보안을 위해 일방향 암호화를 사용하여 고객고유번호 암호화
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|-----------|-----------|--------|-------------|-------------|
| 고객고유번호 | String | 20 | 빈값 가능 | 원본 고객고유번호 |
| 일방향서비스ID | String | 12 | NOT NULL | 암호화 서비스 식별자 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|-----------|-----------|--------|-------------|-------------|
| 암호화고객고유번호 | String | 44 | Encrypted format | 암호화된 고객고유번호 |

- **처리 로직:**
  - 고객고유번호가 빈값인지 확인 (처음 2자리가 공백)
  - 빈값이면 암호화 없이 공백으로 설정
  - 빈값이 아니면 ZUGCDCV 모듈을 사용하여 ASCII 형식으로 변환
  - 서비스ID와 함께 XFAVSCPN 모듈을 사용하여 일방향 암호화 적용
  - 44바이트 암호화 결과 반환
  - 모니터링을 위해 암호화 작업 및 오류 카운트
- **적용된 업무 규칙:** [BR-004-007]

### F-004-003: 레코드암호화및출력처리
- **설명:** 완전한 출력 레코드를 암호화하고 출력 파일에 작성
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|-----------|-----------|--------|-------------|-------------|
| 출력레코드 | String | 1232 | Fixed length | 완전히 포맷된 출력 레코드 |
| 양방향서비스ID | String | 12 | NOT NULL | 암호화 서비스 식별자 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|-----------|-----------|--------|-------------|-------------|
| 암호화출력파일 | File | 1664 bytes per record | Encrypted format | 암호화된 출력 파일 |
| 평문출력파일 | File | 1232 bytes per record | Plain text format | 평문 출력 파일 |

- **처리 로직:**
  - ZUGCDCV 모듈을 사용하여 전체 레코드를 ASCII 형식으로 변환
  - 서비스ID와 함께 XFAVSCPN 모듈을 사용하여 양방향 암호화 적용
  - 암호화된 레코드를 암호화 출력 파일에 작성 (1664바이트)
  - 평문 레코드를 평문 출력 파일에 작성 (1232바이트)
  - 모니터링을 위해 성공한 암호화 및 오류 카운트
- **적용된 업무 규칙:** [BR-004-008]

### F-004-004: 암호화서비스설정조회
- **설명:** 시스템 분류에 따른 암호화 서비스 구성 조회
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|-----------|-----------|--------|-------------|-------------|
| 시스템구분 | String | 3 | NOT NULL | 시스템 환경 분류 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|-----------|-----------|--------|-------------|-------------|
| 일방향서비스ID | String | 12 | NOT NULL | 일방향 암호화용 서비스 ID |
| 양방향서비스ID | String | 12 | NOT NULL | 양방향 암호화용 서비스 ID |

- **처리 로직:**
  - 암호화 서비스 구성을 위해 THKIIK923 테이블 쿼리
  - 그룹회사코드 'KB0'과 신용평가관리코드 'EN'으로 필터링
  - CASE 로직을 사용하여 시스템 분류에 따른 서비스 ID 선택
  - ZAD: KB0KIID01/KB0KIID02 사용, ZAB: KB0KIIB01/KB0KIIB02 사용, ZAP: KB0KIIP01/KB0KIIP02 사용
  - 다른 분류에 대해서는 KB0KIIB01/KB0KIIB02로 기본값 설정
  - 관리구분내용과 세부관리내용을 연결
- **적용된 업무 규칙:** [BR-004-010]

### F-004-005: 체크파일생성
- **설명:** 처리 통계가 포함된 요약 체크 파일 생성
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|-----------|-----------|--------|-------------|-------------|
| 기준일자 | Date | 8 | YYYYMMDD format | 처리 기준 날짜 |
| 현재일자 | Date | 8 | YYYYMMDD format | 현재 처리 날짜 |
| 레코드건수 | Numeric | 10 | NOT NULL | 처리된 레코드 수 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|-----------|-----------|--------|-------------|-------------|
| 체크파일레코드 | String | 28 | Fixed format | 처리 통계가 포함된 요약 레코드 |

- **처리 로직:**
  - 파이프 구분자로 체크 레코드 포맷
  - 기준일자, 현재일자, 총 레코드 수 포함
  - 체크 파일에 단일 요약 레코드 작성
  - NDM 서버 전송 지원에 사용
- **적용된 업무 규칙:** 특정 규칙 없음

### 데이터베이스 테이블

- **THKIPA112**: 기업집단평가정보 - 기업집단차주 데이터의 주요 소스 테이블
- **THKAABPCB**: 고객기본정보 - 고객고유번호 및 분류 데이터
- **THKIIK923**: 신용평가관리정보 - 암호화 서비스 구성 데이터

### 오류 코드

- **오류 집합 FILE_OPERATIONS**:
  - **에러코드**: File Status Codes - 다양한 파일 작업 오류 코드
  - **조치메시지**: "파일 작업 실패" - 표준 파일 오류 처리
  - **사용**: 프로그램 전반의 파일 열기, 닫기, 읽기, 쓰기 작업

- **오류 집합 SQL_OPERATIONS**:
  - **에러코드**: SQLCODE 값 - 데이터베이스 작업 오류 코드
  - **조치메시지**: SQLCODE, SQLSTATE, SQLERRM 세부사항이 포함된 SQL 오류 메시지
  - **사용**: 커서 작업 및 데이터 조회를 포함한 모든 SQL 작업

- **오류 집합 ENCRYPTION_OPERATIONS**:
  - **에러코드**: XFAVSCPN-R-STAT - 암호화 서비스 반환 상태
  - **조치메시지**: 오류 카운팅이 포함된 암호화 오류 처리
  - **사용**: 일방향 및 양방향 암호화 작업

### 기술 아키텍처

- **BATCH Layer**: BIP0013 - JCL 작업 제어가 포함된 주요 배치 처리 프로그램
- **SQLIO Layer**: THKIPA112, THKAABPCB, THKIIK923 테이블을 위한 DB2 데이터베이스 액세스 컴포넌트
- **Framework**: #CRYPTN, #DYCALL, #OKEXIT 매크로 사용이 포함된 COBOL 프레임워크

### 데이터 흐름 아키텍처

1. **입력 흐름**: SYSIN 파라미터 → BIP0013 → 파라미터 검증
2. **데이터베이스 액세스**: BIP0013 → DB2 SQLIO → THKIPA112/THKAABPCB/THKIIK923 테이블
3. **서비스 호출**: BIP0013 → XZUGCDCV (코드 변환) → XFAVSCPN (암호화 서비스)
4. **출력 흐름**: BIP0013 → 암호화 출력 파일 (1664바이트) + 평문 출력 파일 (1232바이트) + 체크 파일 (28바이트)
5. **오류 처리**: 모든 레이어 → 프레임워크 오류 처리 → 메시지 표시 및 프로그램 종료
