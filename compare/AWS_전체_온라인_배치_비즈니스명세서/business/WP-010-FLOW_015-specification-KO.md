## 1. 개요

본 워크패키지는 신용평가 및 고객관리 목적으로 기업집단 합산연결재무제표를 생성하는 포괄적인 배치 처리 시스템을 구현합니다. 시스템은 기업집단 평가 데이터를 처리하고 계열사들의 개별재무제표를 집계하여 다년도에 걸친 합산연결재무제표를 생성합니다.

업무 목적:
- 기업집단 합산연결재무제표 생성
- 계열사 개별 및 연결재무제표 집계
- 포괄적인 재무분석을 위한 다년도 결산년 처리
- 합산재무지표를 통한 신용평가 지원
- 추세분석을 위한 과거 재무데이터 유지
- 당행 및 외부평가기관 재무데이터 처리
- IFRS 및 GAAP 재무제표 양식 처리

시스템은 확장된 다중 모듈 흐름을 통해 데이터를 처리합니다: BIP0028 → RIPA130 → YCCOMMON → YCDBIOCA → YCCSICOM → YCCBICOM → TRIPC130 → TKIPC130 → TRIPC140 → TKIPC140, 기업집단 재무데이터 집계, 개별재무제표 처리, 합산재무제표 생성을 처리합니다.

주요 업무 기능:
- 기업집단 식별 및 선택
- 다년도 재무데이터 처리 (4년 과거 데이터: 현재년도부터 4년 전까지)
- 내부 및 외부 출처로부터 개별/연결재무제표 수집
- 합산재무제표 계산 및 생성
- 연결대상 개별재무제표 관리
- 데이터 무결성 및 감사추적 유지
- IFRS 및 GAAP 회계기준 지원

## 3. 업무 규칙

### BR-010-001: 기업집단선택규칙
- **설명:** 합산연결재무제표 생성 대상 기업집단 선택 규칙
- **조건:** THKIPB110에 기업집단이 존재하고 THKIPC110에 재무제표반영여부 = '0'이고 기업집단처리단계구분 != '6'인 해당 항목이 있으면 처리 대상으로 선택
- **관련 엔티티:** [BE-010-001]
- **예외사항:** 기업집단처리단계구분 = '6' (완료된 처리 단계)인 그룹은 제외

### BR-010-002: 다년도처리규칙
- **설명:** 다년도(4년 범위)에 걸친 재무제표 처리 규칙
- **조건:** 기업집단이 선택되면 4년 전부터 현재 기준년까지 각 년도를 반복하여 재무데이터를 수집하고 합산 처리
- **관련 엔티티:** [BE-010-001, BE-010-002]
- **예외사항:** 4년 전의 경우 데이터 가용성으로 인해 3년 전 계열사 선택 기준 사용

### BR-010-003: 재무제표유형우선순위규칙
- **설명:** 재무제표 유형(연결 vs 개별) 우선순위 규칙
- **조건:** 계열사 처리 시 모기업고객식별자가 공백이면 연결재무제표(IFRS/GAAP) 사용, 모기업고객식별자가 존재하면 개별재무제표만 사용
- **관련 엔티티:** [BE-010-003]
- **예외사항:** 종속회사(모기업고객식별자 보유)는 개별재무제표만 사용

### BR-010-004: 재무제표출처우선순위규칙
- **설명:** 재무제표 데이터 출처 우선순위 규칙
- **조건:** 연결재무제표가 존재하면 양식(IFRS > GAAP)과 출처(한신평 '5' > 크레탑 '6')로 우선순위 결정, 연결재무제표가 없으면 당행 데이터(THKIIK319) 확인, 당행 데이터가 없으면 외부기관 데이터(THKIIMA10)에서 '2' > '3' 우선순위로 사용
- **관련 엔티티:** [BE-010-003]
- **예외사항:** 계산 날짜 범위 내에서 가장 최근 데이터 사용

### BR-010-005: 재무데이터검증규칙
- **설명:** 합산 전 재무데이터 검증 규칙
- **조건:** 재무항목금액이 0이 아니면 합산에 포함, 금액이 0이면 성능 최적화를 위해 처리에서 제외
- **관련 엔티티:** [BE-010-003, BE-010-004, BE-010-005]
- **예외사항:** 0 금액은 유효하지만 효율성을 위해 처리하지 않음

### BR-010-006: 합산재무제표생성규칙
- **설명:** 개별재무제표를 합산하여 합산재무제표 생성 규칙
- **조건:** 결산년의 모든 개별재무제표가 수집되면 재무항목코드와 보고서구분별로 금액을 합산, 기업집단, 결산년, 재무항목별로 GROUP BY, 보고서구분을 '1' + 원래 구분의 두 번째 자리로 설정
- **관련 엔티티:** [BE-010-004, BE-010-005]
- **예외사항:** 합산 시 NULL 값을 0으로 처리, 합산 데이터의 보고서구분 변환

### BR-010-007: 기존데이터정리규칙
- **설명:** 새 데이터 생성 전 기존 합산연결재무제표 데이터 정리 규칙
- **조건:** 기업집단의 합산 프로세스 시작 시 중복 방지를 위해 동일한 기준년의 기존 합산연결재무제표(THKIPC130)와 개별재무제표(THKIPC140) 삭제

### BR-010-008: 날짜범위계산규칙

### BR-010-009: 입력파라미터검증규칙

- **조건:** 배치 시작 시 작업기준년월일이 공백이 아닌지 검증, 공백이면 오류코드 11로 종료

### BR-010-010: 오류처리규칙
- **설명:** 처리 중 다양한 오류 조건 처리 규칙
- **조건:** 오류 발생 시 적절한 리턴코드 설정: 11-19 입력파라미터 오류, 21-29 데이터베이스 오류, 31-39 배치진행 오류, 91-99 파일제어 오류, 그리고 처리 종료

### BR-010-012: 보고서구분변환규칙

### BR-010-013: 계열사수계산규칙

- **관련 엔티티:** [BE-010-002, BE-010-004]

### BR-010-014: 거래로깅규칙

  - '11'과 '17' 사이의 보고서구분으로 재무항목 처리

  - 합산 데이터의 보고서구분을 '1' + 두 번째 자리로 변환
  - 결산년합계업체수를 고객수로 설정
  - 집계된 데이터를 THKIPC130에 삽입
  - 성공적인 처리 후 변경사항 커밋
- **적용된 업무 규칙:** [BR-010-006, BR-010-012, BR-010-013]

### F-010-008: 다년도처리제어기능
- **설명:** 재무제표 생성을 위한 다년도 처리 루프 제어
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 기준년 | Numeric | 4 | YYYY 형식 | 처리를 위한 기준년 |
| 4년전 | Numeric | 4 | YYYY 형식 | 시작년 (4년 전) |
| 기업집단정보 | Structure | Variable | 완전한 그룹 데이터 | 기업집단 세부사항 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 처리년수 | Numeric | 1 | 1-4 | 처리된 년수 |
| 총생성레코드수 | Numeric | 9 | 음이 아닌 수 | 총 합산 레코드 |
| 처리상태 | String | 2 | 상태 코드 | 전체 처리 상태 |

- **처리 로직:**
  - 4년 전부터 기준년까지 반복
  - 각 년도에 대해 계열사를 선택하고 재무제표 처리
  - 4년 전의 특별한 경우 처리 (3년 전 회사 목록 사용)
  - 각 년도의 고객수와 인스턴스수 추적
  - 인스턴스가 발견되면 합산재무제표 생성
  - 모든 년도가 완료될 때까지 처리 계속
- **적용된 업무 규칙:** [BR-010-002, BR-010-013]

### 기술 아키텍처

- **BATCH Layer**: BIP0028 - JCL 작업 제어를 가진 메인 배치 처리 프로그램
- **SQLIO Layer**: 데이터 조회 및 조작을 위한 직접 SQL 작업
- **DBIO Layer**: RIPA130 - THKIPA130 테이블 접근을 위한 데이터베이스 I/O 작업
- **Framework**: YCCOMMON - 거래 제어 및 로깅을 위한 공통 프레임워크

### 데이터 흐름 아키텍처

1. **입력 흐름**: SYSIN → BIP0028 → 파라미터 검증 → 기업집단 선택
2. **데이터베이스 접근**: BIP0028 → SQLIO → 데이터베이스 테이블 (THKIPB110, THKIPC110, THKIIMA61, THKIIMA60, THKIIK319, THKIIMA10, THKIIK616)
3. **처리 흐름**: 기업집단 선택 → 날짜 계산 → 데이터 정리 → 다년도 처리 → 합산 생성
4. **출력 흐름**: 개별재무제표 → THKIPC140 → 합산 → THKIPC130
5. **오류 처리**: 모든 레이어 → 프레임워크 오류 처리 → 사용자 메시지 및 리턴 코드
