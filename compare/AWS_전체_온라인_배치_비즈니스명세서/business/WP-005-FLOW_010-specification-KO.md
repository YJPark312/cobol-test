# 업무 명세서: 기업집단계열사명세

## 문서 관리

- **업무 도메인:** CREDIT

## 1. 개요

이 작업패키지는 신용평가 목적으로 기업집단 계열사 상세정보를 추출하는 배치 처리 시스템을 구현합니다. 시스템은 THKIPB116(기업집단계열사명세) 테이블에서 기업집단 평가 데이터를 처리하고 BT(Business Trust) 리스크 관리 및 일일 신용평가 데이터 제공을 위한 상세한 계열사 정보를 생성합니다.

업무 목적:
- 일일 신용평가를 위한 기업집단 계열사 상세정보 추출
- 각 계열사별 상세 재무정보 제공
- 보안 데이터 전송을 위한 암호화된 출력파일 생성
- 포괄적인 계열사 데이터로 BT 리스크 관리 지원

## 2. 업무 엔티티

|------|-------------|------|----------|------|-------------|-----------|

| 기업집단코드 | String | 6 | NOT NULL | 기업집단등록코드와 그룹코드의 결합 | WK-O-CORP-CLCT-CD | corpClctCd |
| 집단평가작성년 | String | 4 | YYYY 형식 | 집단평가가 작성된 년도 | WK-O-CLCT-VALUA-WRIT-YR | clctValuaWritYr |

## 3. 업무 규칙

### BR-005-001: 기업집단선택기준
- **설명:** 계열사 데이터 추출을 위한 기업집단 선택 기준을 정의
- **조건:** 계열사 데이터 처리 시 그룹회사코드 = 'KB0' AND 시스템최종처리일시 >= 기준일자인 레코드를 선택
- **관련 엔티티:** BE-005-001, BE-005-006
- **예외사항:** 지정된 날짜 범위에서 레코드를 찾을 수 없는 경우, 빈 결과 집합으로 처리 계속

### BR-005-002: 데이터처리일자로직
- **설명:** 입력 매개변수에 기반한 데이터 처리 기준일자 결정
- **조건:** SYSIN 작업기준일자가 제공되면 제공된 날짜 사용, 그렇지 않으면 시스템 현재일자 - 1일 사용
- **관련 엔티티:** BE-005-006
- **예외사항:** 잘못된 날짜 형식이 제공되면 시스템이 오류로 종료

### BR-005-003: 재무데이터계산규칙
- **설명:** 재무 금액의 계산 및 도출 방법을 정의
- **조건:** 재무 데이터 처리 시 부채총계금액 = 총자산금액 - 자본총계금액
- **관련 엔티티:** BE-005-003
- **예외사항:** 계산 결과가 음수가 될 수 있으며 이는 유효한 업무 케이스임

### BR-005-004: 출력파일생성규칙
- **설명:** 적절한 형식과 암호화로 출력파일을 생성하는 규칙을 정의
- **조건:** 출력파일 생성 시 파이프 구분자 형식으로 암호화 및 비암호화 버전 모두 생성
- **관련 엔티티:** 모든 엔티티
- **예외사항:** 암호화 실패 시 오류 카운트를 기록하지만 처리는 계속

### BR-005-005: 데이터검증규칙
- **설명:** 데이터 무결성 및 일관성을 위한 검증 규칙을 정의
- **조건:** 각 레코드 처리 시 모든 필수 필드가 존재하고 허용 범위 내에 있는지 검증
- **관련 엔티티:** 모든 엔티티
- **예외사항:** 잘못된 레코드는 로그에 기록되지만 처리는 계속

### BR-005-006: 암호화서비스선택
- **설명:** 시스템 환경에 기반한 암호화 서비스 결정
- **조건:** 시스템 환경이 'ZAD'이면 'KB0KIID02' 사용, 'ZAB'이면 'KB0KIIB02' 사용, 'ZAP'이면 'KB0KIIP02' 사용, 그 외에는 'KB0KIIB02' 사용
- **관련 엔티티:** BE-005-006
- **예외사항:** 암호화 서비스를 찾을 수 없으면 시스템이 오류로 종료

## 4. 업무 기능

### F-005-001: 처리환경초기화
- **설명:** 파일 핸들 및 시스템 매개변수를 포함한 배치 처리 환경을 초기화
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| SYSIN매개변수 | String | 80 | NOT NULL | 작업일자 및 작업정보를 포함한 시스템 입력 매개변수 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 처리상태 | String | 2 | '00'=성공 | 초기화 상태 코드 |

- **처리 로직:**
  - 작업 제어에서 SYSIN 매개변수 수락
  - 출력파일 열기 (암호화, 비암호화, 체크파일)
  - 처리 기준일자 결정
  - 암호화 서비스 구성 검색
  - 카운터 및 작업 변수 초기화
- **적용된 업무 규칙:** BR-005-002, BR-005-006

### F-005-002: 입력매개변수검증
- **설명:** 입력 매개변수의 정확성 및 완전성을 검증
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 작업기준일자 | String | 8 | YYYYMMDD 형식 | 처리를 위한 기준일자 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 검증결과 | String | 2 | '00'=유효 | 매개변수 검증 결과 |

- **처리 로직:**
  - 작업기준일자가 비어있거나 공백이 아닌지 확인
  - 제공된 경우 날짜 형식 검증
  - 검증 실패 시 오류 상태 설정
- **적용된 업무 규칙:** BR-005-002, BR-005-005

### F-005-003: 계열사데이터추출
- **설명:** 커서 기반 처리를 사용하여 데이터베이스에서 계열사 데이터를 추출
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 기준일자 | String | 8 | YYYYMMDD 형식 | 데이터 선택을 위한 처리 기준일자 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 계열사레코드 | Record Set | 가변 | 다중 레코드 | 계열사 데이터 레코드 집합 |

- **처리 로직:**
  - THKIPB116 및 THKAABPCB 테이블에 대한 커서 열기
  - 그룹회사코드 및 고객식별자로 테이블 조인
  - 그룹회사코드 = 'KB0' 및 고객고유번호구분 = '07'로 필터링
  - 시스템최종처리일시 >= 기준일자로 필터링
  - 데이터 끝까지 순차적으로 레코드 가져오기
- **적용된 업무 규칙:** BR-005-001

### F-005-004: 개별레코드처리
- **설명:** 데이터 변환 및 출력 생성을 포함한 각 계열사 레코드 처리
- **입력:**

| 입력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 원시데이터베이스레코드 | Record | 가변 | NOT NULL | 데이터베이스 쿼리의 단일 레코드 |

- **출력:**

| 출력 | 데이터 타입 | 길이 | 제약조건 | 설명 |
|------|-------------|------|----------|------|
| 형식화된출력레코드 | String | 424 | 고정 길이 | 파일 출력을 위한 형식화된 레코드 |

- **처리 로직:**
  - 데이터베이스 필드를 출력 구조로 매핑
  - 필드 형식화 및 데이터 타입 변환 적용
  - 파생 필드 계산 (예: 부채총계 = 총자산 - 자본총계)
  - 필드 간 파이프 구분자 추가
  - EBCDIC에서 ASCII로 코드 변환 수행
  - 암호화를 적용하여 암호화 버전 생성
- **적용된 업무 규칙:** BR-005-003, BR-005-004, BR-005-005

### 데이터베이스 테이블

- **THKIPB116**: 기업집단계열사명세 - 계열사 재무정보를 포함한 주 데이터 소스
- **THKAABPCB**: 고객기본정보 - 고객식별자 매핑을 통해 법인등록번호를 검색하는 데 사용

### 오류 코드

- **오류 집합 BATCH_PROCESSING**:
  - **에러코드**: '09' - "일반 처리 오류"
  - **조치메시지**: '98' - "비정상 종료"
  - **맞춤메시지**: '99' - "시스템 오류"
  - **사용**: 오류 처리 및 프로그램 종료를 위해 BIP0012.cbl 전체에서 사용

### 기술 아키텍처

- **BATCH Layer**: BIP0012 - JCL 작업 제어를 포함한 주 배치 처리 프로그램
- **SQLIO Layer**: THKIPB116 및 THKAABPCB 테이블 액세스를 위한 데이터베이스 액세스 구성요소
- **Framework**: 공통 처리 및 오류 처리를 위한 YCCOMMON 프레임워크, 코드 변환을 위한 XZUGCDCV, 암호화 서비스를 위한 XFAVSCPN

### 데이터 흐름 아키텍처

1. **입력 흐름**: SYSIN 매개변수 → BIP0012 → 데이터베이스 쿼리
2. **데이터베이스 액세스**: BIP0012 → DB2 SQL → THKIPB116 + THKAABPCB → 결과 집합
3. **서비스 호출**: BIP0012 → XZUGCDCV (코드 변환) → XFAVSCPN (암호화)
4. **출력 흐름**: BIP0012 → 출력파일 (암호화 + 비암호화 + 체크)
5. **오류 처리**: 모든 계층 → YCCOMMON 프레임워크 → 오류 메시지 → 작업 종료
