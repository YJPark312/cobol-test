# 업무 명세서: 기업집단신용평가일일자료제공

## 1. 개요

이 워크패키지는 외부 시스템에 기업집단 신용평가 일일 자료를 제공하는 배치 처리 시스템을 구현합니다. 시스템은 THKIPB110 테이블에서 기업집단 평가 정보를 추출하고, 데이터 변환 및 암호화를 적용하여 지주회사 리스크 관리 시스템으로 전송할 출력 파일을 생성합니다.

업무 목적은 다음과 같습니다:
- 기업집단신용평가일일자료추출
- 전송보안을위한데이터암호화
- 외부시스템용표준출력파일생성
- 감사추적및검증기능제공

## 3. 업무 규칙

### BR-003-001: 그룹회사필터링규칙
- **설명:** KB금융그룹의 레코드만 처리
- **조건:** 기업집단 데이터 처리 시 그룹회사코드 = 'KB0'으로 필터링
- **관련 엔티티:** BE-003-001
- **예외사항:** 예외 없음 - 모든 데이터는 KB금융그룹이어야 함

### BR-003-002: 일자처리규칙
- **설명:** SYSIN 매개변수 또는 현재 시스템 일자에서 처리 기준일 결정
- **조건:** SYSIN 작업일자가 제공되면 SYSIN 일자 사용, 그렇지 않으면 현재 시스템 일자 - 1일 사용
- **관련 엔티티:** BE-003-001
- **예외사항:** SYSIN 일자가 유효하지 않으면(≤ '00000000') 시스템 일자 사용

### BR-003-003: 기업집단코드구성규칙
- **설명:** 등록코드와 그룹코드를 연결하여 기업집단코드 구성
- **조건:** 기업집단 데이터 조회 시 등록코드 + 그룹코드 연결
- **관련 엔티티:** BE-003-001
- **예외사항:** 두 코드 모두 존재하고 null이 아니어야 함

### BR-003-004: 암호화서비스선택규칙
- **설명:** 시스템 유형에 따른 적절한 암호화 서비스 선택
- **조건:** 시스템 유형이 'ZAD'이면 'KB0KIID02' 사용, 'ZAB'이면 'KB0KIIB02' 사용, 'ZAP'이면 'KB0KIIP02' 사용, 그 외는 'KB0KIIB02' 사용
- **관련 엔티티:** BE-003-010
- **예외사항:** 서비스 ID를 찾을 수 없으면 오류로 처리 종료

### BR-003-005: 데이터변환규칙
- **설명:** COBOL 패킹 십진수 필드를 출력용 표시 형식으로 변환
- **조건:** 출력 레코드 작성 시 패킹 십진수를 부호 있는 표시 형식으로 변환
- **관련 엔티티:** BE-003-005, BE-003-006, BE-003-007

- **조건:** 출력 레코드 형식 지정 시 파이프 문자로 필드 구분

### 데이터베이스 테이블

- **THKIPB110**: 기업집단평가기본 - 주 데이터 소스

  - 주요 필드: 그룹회사코드, 기업집단그룹코드, 기업집단등록코드, 평가년월일

### 기술 아키텍처

- **BATCH Layer**: BIP0011 - 기업집단 데이터 추출을 위한 주 배치 처리 프로그램
- **SQLIO Layer**: THKIPB110 및 THKIIK923 테이블에 대한 DB2 데이터베이스 액세스
- **Framework**: 프로그램 인터페이스 및 오류 처리를 위한 YCCOMMON 프레임워크
- **Encryption**: 데이터 보안을 위한 XFAVSCPN 양방향 암호화 모듈
- **Conversion**: EBCDIC에서 ASCII 변환을 위한 XZUGCDCV 코드 변환 모듈

### 데이터 흐름 아키텍처

1. **입력 흐름**: JCL SYSIN → BIP0011 → 매개변수 처리
2. **데이터베이스 액세스**: BIP0011 → DB2 SQLIO → THKIPB110/THKIIK923 테이블
3. **데이터 처리**: 원시 데이터 → 변환 → 코드 변환 → 암호화
4. **출력 흐름**: 처리된 데이터 → 출력 파일 (암호화/평문/체크)
5. **오류 처리**: 모든 계층 → YCCOMMON 프레임워크 → 오류 로깅 → 프로그램 종료
